# Security Policy

## Supported Versions

Currently, the following versions of Sentrix are being supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of Sentrix very seriously. If you discover a security vulnerability within Sentrix, please DO NOT disclose it publicly.

Instead, please send an e-mail to our security team at **sushen.d3v@gmail.com**. All security vulnerabilities will be promptly addressed.

### What to include in your report:
- A description of the vulnerability.
- Steps to reproduce the vulnerability (proof of concept).
- Potential impact of the vulnerability.

We will acknowledge your report within 48 hours, and we aim to provide a timeline for resolution based on the severity of the issue.

## Responsible Disclosure

We ask that you follow these guidelines:
- Give us a reasonable amount of time to fix the issue before publishing it elsewhere.
- Make a good faith effort to avoid violating privacy, destroying data, or interrupting our services.
- Do not exploit the vulnerability for any reason other than to prove it exists.
